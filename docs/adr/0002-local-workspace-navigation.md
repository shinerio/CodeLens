# ADR-0002：本机仓库导航与 Review 工作空间

**状态：** Accepted  
**日期：** 2026-07-18

## 背景

CodeLens 是绑定回环地址的单用户本机应用。启动时要求仓库根目录会阻碍开箱即用，也无法让 Web 用户在不同磁盘和目录之间切换。Review 创建页原先允许自由输入路径与 Git ref，既容易输错，也无法展示可用 Commit 信息；历史 Review 也缺少可恢复的工作空间入口。

## 决策

- 一键启动和默认 CLI 启动不传仓库允许根目录；POSIX 系统从 `/`、Windows 从所有现有盘符暴露目录导航。显式根目录参数继续作为高级收窄选项。
- 该默认模式只允许绑定回环地址，操作系统账户的目录读取权限构成本地安全边界。
- `/api/repositories/browse` 只返回有上限的目录列表和 Git 仓库标记，不返回普通文件或内容；列表逐项验证启动用户的读取/进入权限，无权限或系统特殊目录异常会被跳过，不影响同级可访问目录。
- `/api/repositories/catalog` 使用固定 Git 命令参数返回全部本地/远程分支，以及每页 10 条的 Commit 元数据。UI 只通过下拉框提交服务端返回的 ref。
- Review 记录新增创建时间、仓库显示名和 `deleted_at` tombstone。`GET /api/reviews` 只列出可见工作空间；删除活动 Review 时在同一持久化边界内写入取消意图和软删除标记。
- 前端路由以 `/reviews/new` 和 `/reviews/{task_id}` 表达工作空间；旧 `/runs/{task_id}` 路由暂时兼容。

## 结果

- 用户无需理解启动参数或手工复制路径/ref，即可从 Web 完成仓库和 Git 范围选择。
- 本机可访问范围显著扩大，因此回环绑定和操作系统账户隔离成为不可绕过的安全前提；未来远程部署必须先引入身份、授权和重新收窄的仓库策略。
- Review 删除可撤销其产品可见性并安全停止活动任务，同时保留 Finding、事件和审计数据，不通过物理删除破坏恢复与取证。
