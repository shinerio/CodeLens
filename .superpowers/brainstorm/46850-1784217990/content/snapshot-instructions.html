快照、规则解析与任务状态

所有 reviewer 使用相同输入；范围差异只发生在快照创建阶段，后续流水线完全复用。

四种 ReviewScope

1. 分支 Diff + 未提交merge-base(base_branch, HEAD) → WORKTREE

适合 PR 前自检。包含当前分支提交、staged、unstaged、未忽略的 untracked 文件。

2. 指定 Commit + 未提交base_commit → WORKTREE

把用户选择的 commit 作为基线。非祖先 commit 允许 direct diff，但 UI 明确警告。

3. 仅未提交HEAD → WORKTREE

只审查 staged、unstaged 和未忽略的 untracked 文件，适合提交前快速检查。

4. 全仓扫描WORKTREE tracked + untracked

按模块分片；遵循 ignore/skip；二进制、超大文件、LFS 指针单独标记而非直接发送。

ReviewTask / FixTask 状态机

CREATED→SNAPSHOTTING→PREPARING→REVIEWING→VALIDATING→SYNTHESIZING→COMPLETED / PARTIAL
REVIEW COMPLETED→FIXING IN WORKTREE→VERIFYING→AWAITING APPROVAL或AUTO APPLYING→APPLIED

任意运行态均可进入 CANCELED / FAILED；部分 reviewer 失败时默认生成 PARTIAL 报告，并明确披露缺失维度。
