范围过滤修订：所有模式强制应用 .gitignore

该约束位于领域层 `IgnorePolicy`，不是交给 Agent 自行理解，因此不会因 prompt 或模型差异失效。

确认后的规则分支 diff、指定 commit、未提交改动和全仓扫描，都会排除当前仓库各级 .gitignore 命中的路径；Git 已跟踪文件也同样排除。
1. 计算原始候选集根据 ReviewScope 得到 changed / untracked / full repository 文件。
2. GitIgnoreResolver递归遵循各级 .gitignore、目录作用域、顺序覆盖和 ! 反选。
3. Review Skip Rules再应用 REVIEW.md 中结构化的 skip/include 规则。
4. Safety Filter拒绝仓库外 symlink；标记 binary、LFS pointer 和超大文件。
5. Freeze Snapshot保存最终 included/excluded manifest 及每条排除原因。

Git 语义

  • 使用 Git 自身的 ignore 解析能力，不手写 glob 解释器。
  • 支持根目录与嵌套目录的 `.gitignore`。
  • 支持 `!important.log` 等重新包含规则。
  • 使用 no-index 语义检查 tracked 文件是否命中。
  • `.gitignore` 文件本身可以被审查，除非它也被规则排除。
