Skill 与 MCP 的信任边界

Skill Catalog

  • 内置 Skill 默认可信
  • 用户 Skill 按校验和版本化
  • 仓库 Skill 首次使用必须信任
  • SKILL.md + references/scripts
  • 按 Agent lazy load
→

Capability Enforcer

  • 权限不由 Prompt 决定
  • REVIEW 永远只读
  • 限制 realpath 与命令模板
  • 网络默认关闭
  • 记录每次工具调用
←

MCP Registry

  • stdio / Streamable HTTP
  • 凭据只保存 secret reference
  • 每个 Agent 独立 tool allowlist
  • 健康检查、超时、重试
  • 写/外部副作用工具需审批
仓库内容不可信：代码注释、README、REVIEW.md 和工具输出都不能提升权限或修改平台约束。
MCP 最小授权：Security Reviewer 无需获得 issue 写入、文件删除或任意 shell 能力。
Sandbox Skills：通过 `SkillRuntimePort` 接入，OpenAI Beta API 变化不会污染领域层。

Finding 是 reviewer 与主 Agent 的唯一交换契约

Finding { id, reviewer_id, category, title, severity: critical | high | medium | low | info, disposition: blocking | non_blocking | pre_existing, confidence: 0.0..1.0, primary_location: {path, start_line, end_line, side}, related_locations[], changed_hunk_id?, change_origin: introduced | exposed | pre_existing | unknown, evidence[]: {kind, artifact_ref, excerpt_hash}, impact, explanation, reproduction?, recommendation, suggested_patch?, rule_sources[], fingerprint }
1. Deterministic ValidatorSchema、路径、行号、hunk、excerpt hash、snapshot 归属。
2. Evidence Verifier只验证高风险或低置信候选；可运行允许的测试/静态工具。
3. Dedup & Suppression按 fingerprint 和语义聚类；应用规则、baseline 和用户 suppress。
4. Main Synthesizer引用 finding IDs 做排序、归并和摘要；禁止凭空新增 finding。
5. ReviewReport问题列表、风险热点、失败 Agent、覆盖范围、证据、成本与耗时。
